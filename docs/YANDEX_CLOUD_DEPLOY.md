# Переезд сайта на Yandex Cloud (Object Storage + CDN)

Цель: увести `oneofstudio.ru` с Netlify (чьи IP блокируются в РФ) на Yandex Cloud,
чтобы сайт открывался из России без VPN. Сайт статический, поэтому используем
Object Storage (хранилище файлов) + Cloud CDN (раздача по домену с HTTPS).

## Что нужно от тебя заранее
- Аккаунт Yandex (Yandex ID).
- Банковская карта для платёжного аккаунта (для новых аккаунтов обычно есть
  стартовый грант; статический сайт стоит копейки — единицы рублей в месяц).
- **Доступ к управлению DNS домена `oneofstudio.ru`** — там, где сейчас прописана
  A-запись на Netlify (панель регистратора домена). Без этого доменную часть не
  завершить.

## Файлы для загрузки
Готовый чистый набор (только то, что нужно на хостинге) собран в папке `deploy/`
(в рабочей сессии). Если её нет — залей из проекта ровно это, сохраняя структуру:
```
index.html
style.css
assets/web/*        (webp, mp4)
brand_assets/web/*  (logo.png, favicon.*, apple-touch-icon.png)
```
НЕ загружай: `assets/originals/`, `brand_assets/originals/`, `docs/`, `CLAUDE.md`,
`.claude/`, `temporary screenshots/` и прочие служебные файлы.

---

## Шаг 1. Платёжный аккаунт и каталог
1. Зайди в консоль: https://console.yandex.cloud
2. Создай платёжный аккаунт (Billing), привяжи карту.
3. Каталог (folder) `default` уже есть — работаем в нём.

## Шаг 2. Бакет Object Storage
1. Object Storage → **Создать бакет**.
2. Имя: **`oneofstudio.ru`** (ровно как домен — так удобнее и совместимо с CDN).
3. Макс. размер: ограничь, например, 1 ГБ.
4. Доступ:
   - «Доступ на чтение объектов» → **Публичный**.
   - «Доступ на чтение списка объектов» → можно оставить Ограниченный.
5. Создай бакет.

## Шаг 3. Загрузка файлов
1. Открой бакет → **Загрузить**.
2. Перетащи содержимое папки `deploy/`: файлы `index.html`, `style.css` и
   **папки** `assets` и `brand_assets` целиком (консоль сохранит вложенность —
   ключи объектов должны получиться `assets/web/hero.webp` и т.д.).
3. После загрузки проверь у пары файлов «Тип контента» (Content-Type):
   - `index.html` → `text/html`
   - `style.css`  → `text/css`
   - `.webp`      → `image/webp`
   - `.svg`       → `image/svg+xml`
   - `.mp4`       → `video/mp4`
   Обычно определяется автоматически по расширению; если у чего-то стоит
   `application/octet-stream` — поправь вручную (Свойства объекта).

## Шаг 4. Включить хостинг сайта
1. Бакет → вкладка **«Веб-сайт»** → режим **Хостинг**.
2. Главная страница: `index.html`
3. Страница ошибки: `index.html` (для одностраничника проще так; можно сделать
   отдельную 404).
4. Сохрани. Появится «website endpoint» вида:
   `oneofstudio.ru.website.yandexcloud.net`
5. Проверь: открой этот адрес в браузере — сайт должен отобразиться. Это
   промежуточная проверка ещё до подключения домена.

## Шаг 5. TLS-сертификат (HTTPS)
1. **Certificate Manager** → создать сертификат **Let's Encrypt** для
   `oneofstudio.ru` (и добавь `www.oneofstudio.ru`, если нужен www).
2. Способ проверки владения — **DNS CNAME**: сервис выдаст запись, которую надо
   добавить в DNS домена. Добавь её у регистратора и дождись валидации.
   (Сертификат бесплатный, продлевается автоматически.)

## Шаг 6. CDN-ресурс
1. **Cloud CDN** → **Создать CDN-ресурс**.
2. Источник (origin):
   - Тип — **Бакет** → выбери бакет `oneofstudio.ru`
     (или «Своя конфигурация» с origin-хостом
     `oneofstudio.ru.website.yandexcloud.net`, протокол HTTP).
3. Доменные имена для раздачи (personal domain): **`oneofstudio.ru`**
   (и `www.oneofstudio.ru`, если используешь).
4. Сертификат: выбери выпущенный на шаге 5.
5. Протоколы: включи **редирект HTTP → HTTPS**.
6. Создай ресурс. CDN выдаст CNAME-цель вида `cl-xxxxxxxx.edgecdn.ru`
   (или `*.gcdn.co`). Запиши её — понадобится для DNS.

## Шаг 7. DNS (самый важный шаг переключения)
Голый домен `oneofstudio.ru` (apex) по стандарту DNS **нельзя** указать через
обычный CNAME. Варианты:

**Вариант A (рекомендуется) — перенести DNS-зону в Yandex Cloud DNS:**
1. Cloud DNS → создать публичную зону для `oneofstudio.ru`.
2. У регистратора смени NS-серверы домена на выданные Yandex Cloud DNS.
3. В зоне создай запись типа **ANAME/ALIAS** для `oneofstudio.ru` → CNAME-цель CDN
   (Yandex Cloud DNS умеет ALIAS на apex).
4. Для `www` — обычный **CNAME** → та же цель CDN.

**Вариант B — остаться у текущего регистратора:**
- Если его панель поддерживает **ALIAS/ANAME** на apex — используй его на
  CNAME-цель CDN. `www` — обычный CNAME.
- Если ALIAS не поддерживается — придётся использовать Вариант A.

Перед переключением полезно **заранее снизить TTL** старой A-записи (напр. до
300 сек), чтобы переключение прошло быстрее.

Удали старую A-запись на Netlify (`75.2.60.5`) после того, как новая заработает.

## Шаг 8. Проверка
1. Дождись обновления DNS (от минут до нескольких часов по TTL).
2. Открой `https://oneofstudio.ru` — должен грузиться через CDN, с валидным HTTPS.
3. Проверь **с российских точек** без VPN: https://ping.pe/oneofstudio.ru или
   https://2ip.ru/ping/ — таймаутов быть не должно.
4. Проверь favicon, картинки, видео, ссылки на контакты.

## После переезда
- Netlify можно отключить (удалить сайт/деплой), чтобы не платить и не путаться.
- На будущее: чтобы обновлять сайт, перезаливай изменённые файлы в бакет и, при
  необходимости, делай «сброс кэша» (purge) в CDN для обновлённых путей.

## Приложение: загрузка через CLI (по желанию, быстрее консоли)
Object Storage совместим с S3. Через AWS CLI:
1. В консоли Yandex создай **сервисный аккаунт** с ролью `storage.editor` и
   выпусти для него **статический ключ доступа** (Access Key + Secret Key).
2. Настрой профиль (ключи вводишь ты, не передавай их никому):
   ```
   aws configure --profile ycloud
   # Access Key / Secret Key — из сервисного аккаунта
   # region: ru-central1
   ```
3. Синхронизируй чистую папку в бакет:
   ```
   aws --profile ycloud --endpoint-url=https://storage.yandexcloud.net \
       s3 sync ./deploy s3://oneofstudio.ru/ --delete
   ```
4. Проверь Content-Type у `.webp`/`.svg` после заливки (CLI иногда ставит
   `application/octet-stream`); при необходимости перезалей с явным
   `--content-type`.

## Замечание про шрифты (необязательно)
Сайт тянет шрифты с Google Fonts (`fonts.googleapis.com`). Обычно в РФ они
доступны, но бывают подтормаживания. Если захочешь полную автономность — можно
скачать Montserrat/Oswald/Saira, положить в `assets/web/fonts/` и подключить
локально через `@font-face`. Это отдельная небольшая задача, к блокировке
Netlify отношения не имеет.
